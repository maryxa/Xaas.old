1 # XaLib
2 c++ application framework
3 
4 Designed to get easier developing of web apps using the most advanced patterns:
5 
6  - 100% object-oriented coding: increased understanding, maintenance and evolution of the code.
7  - MVC model-view-controlled implementation: split page output / code algorithm.
8  - Modular, scalar, extensible design, making each component independent.
9  - HTML5-CSS3 compliance: the view module (XSLT) implements HTML5, making form generation and validation easier.
10  - Control: web pages accessible from both CGI interface and Web Services.
11  - Security: automatic security checks performed on libraries’ calling.
12  - C++ standard: our framework is based on C++11 aka C++0x and is fully portable over different operating systems.
13  - OS-Platform: Windows, Linux, Mac OS X, Raspberry
14  - Memory management with SmartPointer
15  - Configration: One Configuration File
16 
17 Built-in operation’s library with no-coding security checks:
18 
19  - HTTP requests: providing API to read parameters from GET or POST HTTP requests performing internal security checks.
20  - Database abstraction layer: joining communications between code and databases provides a single API to execute and manage different operations, giving independence to the code from the database engine.
21  - Database qeury: collections support for Result Set (via std:map).
22  - Super Globals Variables: REQUEST, RESPONSE and Web Services.
23  - Databse Partitioning: multiples database configuration (Session and authentication, Read, Write).
24  - XML manipulation: ready-to-go methods to manipulate XML objects performing various operations as converting strings and/or files to DOM, counting nodes, evaluating XPath expressions etc.
25  - XSLT parsing: platform-independent methods to parse XML strings, files or documents, setting XSL values which results into HTML text.
26  - Client-server side encryption to protect HTTP communication from the client to the server, including SHA, AES, DES via OpenSSL.
27  - Session Management: the framework comes with methods to manage single-sign-on sessions and cookies between Apps sharing the same framework.
28  - Detailed and flexible Authentication-Permissions-ACL: unlimited users, GUI to set role-based permissions and ACL.
29  - Translations: unlimited languages and GUI-guided translation, so to ease internationalisation and localisation.
30  - Dropdown-boxes management: fully GUI-manageable option list in dropdown boxes stored in a single database table.
31  - Logging: built-in methods defining different logging levels.
32  - Outbound calls with Curl and sockets: handling of Curl calls to external websites managing responses or TCP/IP sockets with external servers.
33  - Mail: ready-to-use methods to send emails.
34  - Error and info page: automatic methods to generate error and info messages.
35  - Character encoding: multiple UTF-8 characters dealing with XML entities and special characters, simplifying conversion to or from URL or Base64, etc.
36  - Time Functions: ready-to-use methods to manipulate dates
37  - Currency Management
38  - Casting: Ready to use casting method
39  - Xml Web Services: ready to go Controller and Interfaces to publish Xml Web Services
40 
41 Raspberry PI:
42 - GPIOs : ready-to-use methods to control GPIO
43 - UART : ready-to-use methods for UART
44 
45 Ease of learning, featuring:
46 
47  - Design patterns: that means the framework is quick, easy to learn and ready to use.
48  - Detailed knowledge of the system not required.
49  - Selective learning: thanks to modular design and model-view-controlled patterns, programmers would need to learn only selected functions relative to their project goal, scope and role.
